## Advanced Database Queries with Diesel

*   **Dynamic Filtering**: It has optional filters for `account_id`, `activity_type`, and `asset_id`. This is achieved by building the query step-by-step, only adding a `filter` clause if the corresponding filter is present.
*   **Pagination**: It uses Diesel's `limit` and `offset` methods to implement pagination.
*   **Dynamic Sorting**: It allows sorting the results by different columns, in either ascending or descending order.

Here's a simplified snippet of the code:

## The Brains of the Operation: Portfolio Performance Calculation

The `src-core/src/portfolio/performance` module is where the real magic happens. This is where Wealthfolio calculates the performance of your portfolio, using industry-standard methods like Time-Weighted Return (TWR) and Money-Weighted Return (MWR).

The `calculate_account_performance` function is the heart of this module. It takes a history of daily account valuations and calculates a wide range of performance metrics. Here's a simplified overview of how it calculates the TWR:

1.  It iterates through the daily valuations in chronological order.
2.  For each day, it calculates the daily return, taking into account any cash flows (deposits or withdrawals). The formula for the daily TWR is: `(end_value / (start_value + cash_flow)) - 1`.
3.  It then chains these daily returns together by multiplying them, to get the cumulative TWR for the entire period.

This is a complex calculation, but by using Rust's powerful features like iterators and its precise `Decimal` type, Wealthfolio is able to perform these calculations accurately and efficiently.

## Robust Error Handling

A financial application needs to be robust and reliable. That's why Wealthfolio has a comprehensive error handling strategy, built around Rust's `Result` enum and the `thiserror` crate.

The `src-core/src/errors.rs` file defines a central `Error` enum that represents all the possible errors that can occur in the `src-core` crate.

```rust
// src-core/src/errors.rs

#[derive(Error, Debug)]
pub enum Error {
    #[error("Database operation failed: {0}")]
    Database(#[from] DatabaseError),

    #[error("Input validation failed: {0}")]
    Validation(#[from] ValidationError),

    #[error("Market data operation failed: {0}")]
    MarketData(#[from] MarketDataError),

    // ... other error variants
}
```
The `#[from]` attribute, provided by the `thiserror` crate, automatically generates `From` implementations for the specified error types. This makes it easy to convert from a specific error type (like `DatabaseError`) to the general `Error` enum, which is what makes the `?` operator so powerful.

For example, in a function that returns a `Result<T, Error>`, you can write:

```rust
let connection = pool.get()?; // This will automatically convert a `PoolError` into an `Error::Database`
```

This strategy makes the error handling code clean, concise, and robust.
