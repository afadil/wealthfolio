# Under the Hood: Wealthfolio's Rust-Powered Financial Engine

Wealthfolio is a desktop application that helps you track your investments. But have you ever wondered what's going on under the hood? How does it store your financial data? How does it calculate your portfolio's performance? The answer to these questions lies in Wealthfolio's powerful backend, written in Rust.

In this blog post, we'll take a deep dive into the `src-core` of Wealthfolio. We'll explore how it uses Rust and the Diesel ORM to manage financial data, and how it performs complex calculations for portfolio performance and analysis. This post will give you a behind-the-scenes look at the engine that powers Wealthfolio.

## The Heart of the Application: The `src-core` Crate

The `src-core` directory is the heart of Wealthfolio. It's a Rust crate that contains all the core business logic of the application. It's responsible for everything from managing the database to calculating complex financial metrics.

Here's a look at the structure of the `src-core` crate:

```
src-core/
├── src/
│   ├── accounts/
│   ├── activities/
│   ├── assets/
│   ├── db/
│   ├── fx/
│   ├── goals/
│   ├── limits/
│   ├── market_data/
│   ├── portfolio/
│   ├── secrets/
│   ├── settings/
│   ├── lib.rs
│   └── schema.rs
├── Cargo.toml
└── diesel.toml
```

Each module in `src-core` has a specific responsibility:

*   **`accounts`**: Manages user accounts.
*   **`activities`**: Handles financial activities like trades, dividends, and fees.
*   **`assets`**: Manages financial assets.
*   **`db`**: Contains the database connection and migration logic.
*   **`fx`**: Handles currency exchange rates.
*   **`goals`**: Manages financial goals.
*   **`limits`**: Manages contribution limits.
*   **`market_data`**: Fetches and manages market data.
*   **`portfolio`**: Contains the logic for calculating portfolio performance, holdings, and valuations.
*   **`secrets`**: Manages encrypted secrets.
*   **`settings`**: Manages application settings.
*   **`schema.rs`**: This file is automatically generated by Diesel and defines the database schema.

## Talking to the Database: Diesel and SQLite

Wealthfolio uses SQLite as its database, which is a great choice for a desktop application because it's lightweight and doesn't require a separate server. To interact with the database, Wealthfolio uses [Diesel](https://diesel.rs/), a popular ORM (Object-Relational Mapper) and query builder for Rust.

### Database Migrations

Diesel's migration system makes it easy to manage changes to the database schema over time. The migrations are located in the `src-core/migrations` directory. Each migration has an `up.sql` file for applying the migration and a `down.sql` file for reverting it.

### Querying with Diesel

Here's an example from `src-core/src/accounts/accounts_repository.rs` that shows how Diesel is used to query the database for all accounts:

```rust
// src-core/src/accounts/accounts_repository.rs

use diesel::prelude::*;
use crate::schema::accounts;

pub fn get_all_accounts(
    conn: &mut SqliteConnection,
) -> Result<Vec<Account>, DbError> {
    let results = accounts::table
        .load::<Account>(conn)?;
    Ok(results)
}
```

As you can see, Diesel provides a type-safe way to write queries. The `accounts::table` module is generated from the `schema.rs` file, and the `.load::<Account>(conn)` method executes the query and deserializes the results into a `Vec<Account>`. This makes the code easy to read, write, and maintain.

## The Brains of the Operation: Portfolio Calculation

The `src-core/src/portfolio` module is where the magic happens. This is where Wealthfolio calculates all the important metrics for your portfolio, such as holdings, performance, and valuations.

The logic in this module is complex, as it has to take into account a wide range of factors, including:

*   **Different types of activities**: Trades, dividends, fees, and transfers all have to be handled differently.
*   **Multiple currencies**: The module has to handle currency conversions and exchange rate fluctuations.
*   **Time-weighted returns**: To accurately calculate performance, the module uses the time-weighted rate of return method, which accounts for the timing of cash flows.

While the full implementation is too detailed to cover in this blog post, the general idea is that the portfolio calculation service iterates through all the financial activities in chronological order, and for each activity, it updates the state of the portfolio. This includes updating the quantity of each holding, the cost basis, and the cash balance.

This iterative approach allows Wealthfolio to accurately reconstruct the state of your portfolio at any point in time, which is essential for generating historical performance charts and other analytics.

## Bridging the Gap: Frontend-Backend Communication

So, how does the React frontend get access to the data from the Rust backend? The answer is Tauri commands.

Tauri provides a simple and secure way to call Rust functions from JavaScript. In Wealthfolio, the `src-tauri/src/commands` directory contains all the Tauri commands that are exposed to the frontend.

Here's an example from `src-tauri/src/commands/account.rs` that shows how the `get_all_accounts` function from the `src-core` is exposed as a Tauri command:

```rust
// src-tauri/src/commands/account.rs

use wealthfolio_core::accounts::IAccountsService;

#[tauri::command]
pub async fn get_all_accounts(
    accounts_service: State<'_, Arc<dyn IAccountsService>>,
) -> Result<Vec<Account>, CommandError> {
    let accounts = accounts_service.get_all_accounts()?;
    Ok(accounts)
}
```

The `#[tauri::command]` macro automatically generates the necessary JavaScript bindings to call this function from the frontend. The `accounts_service` is a dependency that is injected by Tauri's dependency injection system. This service is an implementation of the `IAccountsService` trait from `src-core`.

This clean separation of concerns between the core logic in `src-core` and the command handlers in `src-tauri` makes the code easy to test and maintain.

## Conclusion: Rust as the Engine of a Financial App

Wealthfolio's `src-core` is a testament to the power of Rust for building financial applications. By leveraging Rust's safety, performance, and rich ecosystem of libraries like Diesel, Wealthfolio is able to build a reliable and maintainable backend that can handle complex financial calculations with ease.

The clear separation between the core logic and the frontend, connected by the secure bridge of Tauri commands, is a great example of how to build a modern desktop application. We hope this deep dive into the `src-core` of Wealthfolio has given you a better understanding of what goes on under the hood of this powerful investment tracker.
